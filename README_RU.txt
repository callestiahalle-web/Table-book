Синхронизация «Мне нравится»
— Основным источником списка является Supabase, уже подключённый к проекту.
— Избранное хранится в public.user_app_state → app_state → favorites для текущего user_id.
— Используются существующие авторизация, таблица user_app_state и RLS-политики проекта.
— Новую таблицу и новые Supabase URL/anon key создавать не нужно.
— После входа список автоматически загружается из Supabase.
— Добавление и удаление сердечка автоматически записывается в Supabase.
— Изменения с другого устройства подхватываются через Supabase Realtime, а также при возвращении во вкладку и открытии раздела «Мне нравится».
— Локальное хранилище используется только как персональный офлайн-кэш. После восстановления интернета изменения отправляются в Supabase.
— Старый локальный список table_book_favorites_v1 переносится в Supabase один раз, только если в облачном профиле ещё нет поля favorites.
— Кэш разделён по user_id, поэтому избранное разных аккаунтов не смешивается.

Требования к Supabase
В проекте должна существовать таблица public.user_app_state со столбцами:
  user_id uuid
  app_state jsonb
  my_recipes jsonb
  updated_at timestamptz

Это уже соответствует файлу supabase_setup.sql текущего проекта. Поле favorites хранится внутри JSONB app_state, поэтому SQL-миграция не требуется.

Как установить обновление
1. Сделайте резервную копию текущей папки приложения.
2. Скопируйте содержимое этого архива в корневую папку Table book.
3. Замените существующий index.html новым файлом.
4. Добавьте или замените файлы:
   css/tablebook-hotfix.css
   js/tablebook-hotfix.js
5. Не удаляйте существующие css/style.css, js/app.js, папку assets и файлы с рецептами.
6. Опубликуйте изменения.
7. Обновите страницу с очисткой кэша: Ctrl+F5.
8. Откройте страну: сначала должны появиться блоки типов блюд, а после выбора типа — рецепты этой категории.
9. Войдите в аккаунт и отметьте блюдо сердечком. После обновления страницы оно должно остаться в «Мне нравится».

Совместимость
Обновление работает дополнительным слоем поверх текущих style.css и app.js. База рецептов, пользовательские рецепты, календарь, профиль и существующая авторизация Supabase не заменяются.
